knot has two command-line interfaces, each documented separately:

- [knot](knot/) - The main binary run from your machine or as the server: spaces, stacks, templates, scripts, forwarding, and administration.
- [knot agent](agent/) - The agent binary that runs inside a space: lifecycle, events, methods, port forwards, tunnels, and run-script.
